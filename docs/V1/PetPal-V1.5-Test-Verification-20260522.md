## 三、时间窗口与任务排序验证（对应场景 2, 3, 9）

### 测试目标
验证任务根据时间段正确显示 `现在去做 / 今天稍后 / 已完成`，并按优先级排序

### 测试步骤

#### 3.1 早晨 08:00 测试

**前提**: 系统时间设置为 08:00

| 步骤 | 操作 | 预期结果 |
|-----|------|---------|
| 1 | 打开小程序，进入 `任务` tab | - |
| 2 | 查看 `现在去做` 区域 | 包含 `刷牙`（时间段 07:00-09:00） |
| 3 | 查看 `今天稍后` 区域 | 包含 `吃饭 / 按时睡觉 / 读绘本 / 整理玩具 / 洗澡` |
| 4 | 查看宠物首页提示 | 文案包含 `先去刷牙吧` |
| 5 | 点击 `刷牙` 任务 | 可正常提交打卡 |

#### 3.2 晚上 19:45 测试

**前提**: 系统时间设置为 19:45

| 步骤 | 操作 | 预期结果 |
|-----|------|---------|
| 1 | 打开小程序，进入 `任务` tab | - |
| 2 | 查看 `现在去做` 区域 | 包含 `读绘本 / 洗澡 / 整理玩具`（根据时间段判断） |
| 3 | 查看任务排序 | `ready` 任务优先，`overdue` 次之，`upcoming` 最后 |
| 4 | 查看宠物首页提示 | 文案匹配当前时间段的最优先任务 |

#### 3.3 跨日重置测试

| 步骤 | 操作 | 预期结果 |
|-----|------|---------|
| 1 | 在 05-22 完成 `刷牙` 任务（审核通过） | 任务显示在 `已完成` |
| 2 | 修改系统日期为 05-23，重新打开小程序 | - |
| 3 | 查看 `任务` tab | `刷牙` 重新出现在 `现在去做` 或 `今天稍后` |
| 4 | 查看 `checkins` 历史 | 05-22 的记录仍保留 |
| 5 | 查看积分 | 昨日奖励的积分保留 |

#### 3.4 时间段边界测试

| 步骤 | 操作 | 预期结果 |
|-----|------|---------|
| 1 | 设置时间为 09:00（刷牙结束时间） | `刷牙` 显示为 `overdue` 或 `现在去做`（边界内） |
| 2 | 设置时间为 09:01 | `刷牙` 显示为 `overdue` |
| 3 | 设置时间为 06:59（刷牙开始前） | `刷牙` 显示为 `upcoming` |

### 验证标准

- [ ] 早晨 08:00，`刷牙` 正确显示在 `现在去做`
- [ ] 晚上 19:45，`读绘本 / 洗澡 / 整理玩具` 根据时间窗口正确排序
- [ ] 跨日后，昨日完成的任务重新可用，历史记录保留
- [ ] 时间段边界（开始/结束时间）处理正确

| 4 | 查看提示文案 | 显示抚摸反馈文案 |
