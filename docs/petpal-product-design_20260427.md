# PetPal 产品方案设计 v0.2
**Author**: Alex（产品经理智能体）
**Date**: 2026-04-27
**Status**: 方案迭代中
**定位确认**：习惯养成 | 目标用户：4-7岁

---

## 一、问题陈述

**家长痛点**：4-7岁孩子做事没动力——"快去刷牙""把饭吃完"说了100遍还是磨蹭。威逼利诱都试过，效果不持久，家长疲了。

**为什么现有方案失效**：

| 方式 | 为什么失效 |
|-----|---------|
| 说教催促 | 孩子当耳旁风，重复N遍无效 |
| 物质奖励（买玩具） | 效果递减，要的越来越多 |
| 小红花/积分墙 | 和孩子没情感连接，贴完就忘 |
| 没收手机惩罚 | 短期有效，但亲子关系受损 |

**根本原因**：这些方式都是**外部激励**，和孩子当下的体验没有连接。奖励遥远，惩罚冷淡，孩子没有内在动力。

**PetPal 的解法**：

> 一个会饿、会难过、会对孩子有"反应"的宠物。孩子的每个好习惯都在喂养它、让它开心——**宠物和孩子共享情绪**，好习惯变成了照顾"家人"而不是完成任务。

## 三、北极星指标

| 指标 | 定义 | 当前假设基线 | 上线60天目标 |
|-----|------|-----------|------------|
| **D7留存** | 连续7天打开小程序的孩子数/新增用户数 | 待测（预计20%） | ≥ 40% |
| **任务完成率** | 当日完成任务数/当日应完成任务数 | 待测（预计50%） | ≥ 70% |
| **家长周活** | 当周至少进入1次家长模式的家长数/总用户数 | 待测（预计30%） | ≥ 50% |

**为什么用D7留存做北极星**：孩子有没有"想要"这个宠物，是唯一重要的事。完成任务率高但留存低，说明孩子在帮家长做，不是在为自己养。

## 四、核心用户故事

### 孩子视角

**小美，5岁，中班**

"我喜欢小狗，小狗饿了的时候我会心疼它。妈妈说完成任务小狗就能吃东西，我就要每天都完成任务，让小狗开开心心的。"

**核心体验路径**：
```
打开小程序 → 看到小狗饿了/不开心 → 心疼 → 去完成任务 → 打卡成功 → 积分到手 → 喂小狗 → 小狗开心 → 孩子满足 → 下次还来
```

### 家长视角

**小美妈妈，职场女性，32岁**

"以前每天催她刷牙吃饭，嗓子都哑了。现在她自己知道要完成任务，不然小狗会饿。我只需要偶尔审核一下，省心多了。"

**核心体验路径**：
```
收到提醒"有任务待审核" → 点开小程序 → 一键通过 → 看到本周完成情况 → 满意地退出
```

## 五、功能优先级

### 🔴 P0 — V1 MVP（4-6周）

**目标**：跑通"任务→积分→宠物互动"核心闭环，1种宠物，1个孩子。

#### 1. 宠物状态系统（核心）
- **饱食度**（0-10整数）：每4小时自然-1；喂食+3（消耗1分）
- **快乐值**（0-10整数）：每6小时自然-1；完成任务+1；玩耍+2（消耗1分）
- **视觉表情**：超级开心/正常/难过/生病四级，纯图标，孩子靠看脸色判断
- **宠物类型**：V1只做小狗（摇尾巴最直观，4-7岁孩子秒懂）

#### 2. 任务系统
- **预设6个任务**（V1内置，无需家长配置）：
  - 🪥 起床刷牙
  - 🍚 吃完饭
  - 🛏️ 按时睡觉
  - 📚 读绘本
  - 🧹 整理玩具
  - 🚿 洗澡
- **打卡方式**：默认"家长确认"模式，防止作弊
- **积分获取**：完成+1分，连续7天额外+1分（连续奖励）

#### 3. 宠物互动
- **喂食**：消耗1分，饱食度+3，动画反馈
- **玩耍**：消耗1分，快乐值+2，动画反馈
- **抚摸**：免费，快乐值+1（日常小互动，不消耗积分）

#### 4. 家长模式
- **入口**：孩子界面右上角小锁图标
- **密码**：4位数字，第一次设置
- **功能**：
  - 查看/审核待打卡记录（一键通过）
  - 开启/关闭每个任务的"需要家长确认"
  - 查看本周任务完成率（简单图表）
  - 修改任务积分（高级选项）

#### 5. 宠物成长
- **3阶段**：蛋→幼崽→成长期
- **升级条件**：累计积分达到阈值（幼崽需20分，成年需60分）
- **阶段奖励**：新外观 + 新动作动画

## 六、用户旅程详细设计

### 首次打开（孩子）

```
第1步：欢迎动画（小狗在屏幕里跑来跑去）→ "你想养一只小狗吗？"
第2步：选颜色/名字（预设5个选项：旺财/小黄/豆豆/团团/小贝）
第3步：宠物"出生"动画（从蛋里跳出来）→ 立即进入主界面
→ 总时长：约60秒，0文字依赖，靠动画和语音引导
```

### 孩子日常打开流程（不超过20秒）

```
打开小程序
  → 直接进入宠物主界面
  → 宠物状态一目了然（饿了？开心？表情+音效）
  → 宠物说"我好饿"（语音提示）
  → 今日任务按钮（红色圆点提示"有新任务"）
  → 点击任务 → 打卡
  → 积分+动画（"刷牙成功 +1分"）
  → 积分够了吗？→ 去喂食 → 宠物开心
  → 完成闭环，满意退出
```

### 家长首次进入

```
入口：孩子界面右上角小锁
  → 设置4位密码
  → 看到预设任务列表（默认全开，可关闭）
  → 看到本周完成情况摘要（3个数字：完成X个/本周Y天/连续Z天）
  → 完成
```

### 家长日常使用

```
收到微信服务通知"有2个任务待确认"
  → 打开小程序 → 家长模式
  → 看到待审核列表（一键全通过）
  → 顺便看一眼本周报告
  → 退出
→ 整个流程不超过30秒
```

## 七、宠物状态设计（4-7岁适配）

### 状态值简化逻辑

**为什么只保留2个值（饱食度+快乐值）**：
- 4-7岁孩子能理解"饱"和"开心"，理解不了"健康值""经验值"
- 数字范围0-10，孩子能比较大小（8比5大）
- 衰减速度设置"宽松模式"：默认每4/6小时减1，家长可调

### 表情系统（纯视觉）

```
超级开心（8-10）：🐕 蹦跳 + 闪金光 + 摇尾巴音效
正常（5-7）：🐕 随机小动作（转圈/伸懒腰）
难过（2-4）：🐕 耷拉耳朵 + 眼睛下垂 + 呜咽音效
生病（0-1）：🐕 躺着 + 眼泪 + 微弱呼吸动画
           → 点击任意道具即可治愈（无死亡惩罚）
```

### 宠物语言（语音泡泡）

不需要孩子认字，靠语音：

| 状态 | 小狗会说（语音播放） |
|-----|---------------|
| 饱食度=2 | "我好饿……肚子咕咕叫" |
| 饱食度=8 | "我吃饱啦！好幸福～" |
| 快乐值=2 | "我有点不开心……" |
| 快乐值=8 | "我好开心！你是最好的主人！" |
| 完成打卡 | "太棒了！谢谢你！" + 摇尾巴 |
| 喂食后 | "好吃！谢谢你！" + 舔嘴动画 |

## 八、习惯养成机制设计

### 核心闭环

```
触发（宠物饿了/不开心）
  → 动机（孩子心疼宠物）
  → 行为（完成任务→打卡）
  → 奖励（积分+宠物开心反馈）
  → 强化（宠物变开心，孩子有成就感）
  → 习惯形成（每天自动打开）
```

### 连续打卡机制

| 连续天数 | 奖励 | 目的 |
|--------|-----|------|
| 3天 | 成就徽章"坚持3天" | 第一次建立连续概念 |
| 7天 | 成就徽章"一周小明星" + 额外1分 | 真正的习惯里程碑 |
| 30天 | 成就徽章"习惯达人" + 宠物新外观 | 长期坚持的认可 |

### 家长激励设计

**家长的核心诉求**：孩子主动做，不用催。

**解决方案**：家长不需要每天操作，只需要：
1. 第一次设置好任务（5分钟，终身受益）
2. 偶尔审核打卡（30秒/天）
3. 查看周报（可选）

**关键设计**：家长模式的"审核"按钮要大、要醒目、要一键完成——减少家长的摩擦，就是增加家长持续使用的概率。

## 十、风险与应对

| 风险 | 影响 | 应对 |
|-----|------|------|
| 孩子沉迷 | 高 | 默认关闭声音，每次打开显示"今天陪小狗X分钟"，家长可设置每日上限 |
| 数据丢失 | 高 | V1设置页提供"导出数据"按钮，导出为txt存储到本地 |
| 审核流程家长懒得用 | 高 | ① 默认开启自动通过（家长可关闭）；② 推送通知提醒；③ 30天内未审核则自动通过 |
| 孩子自己给自己打卡 | 高 | V1默认所有任务需要家长确认，不提供关闭选项 |
| 个人主体审核被拒 | 中 | 选择"工具>生活"类目，规避"教育"类目的资质要求 |
| 宠物生病孩子情绪崩溃 | 低 | 无死亡机制，任意道具1分即可治愈，生病状态是"需要被照顾"而非"要死了" |

## 十一、竞品分析

### 竞品概览

| 产品 | 类型 | 目标用户 | 核心机制 | 主要缺陷 |
|-----|------|---------|---------|---------|
| **子约-培养儿童好习惯** | 微信小程序 | 儿童（年龄未明确） | 星星奖励→兑换礼物 | 无宠物，无情感连接；礼物兑换依赖家长配合；纯积分墙逻辑 |
| **宝宝巴士·习惯养成** | App | 1-5岁 | 故事互动+角色引导（妙妙） | 纯游戏内容，无宠物；需要下载；不连接现实任务 |
| **宝宝巴士·习惯班** | App | 4-5岁 | 场景互动+家长控制 | 同上；VIP付费模式；与现实习惯养成脱节 |
| **成长守护**（腾讯） | 微信小程序 | 家长 | 游戏时长管控+数据共享 | 管控工具而非养成工具；定位完全不同 |
| **ChoreMonster**（美国） | App | 4-14岁 | 家务积分→礼物兑换 | 积分墙模式，无宠物；礼物兑换是家长设置的外部奖励 |
| **Grit Habit Tracker** | App | 成年人/家庭 | 游戏化习惯追踪 | 非儿童专属；无宠物元素 |

### 赢家和输家分析

**做得好的：**
- **宝宝巴士**：IP可爱，动画专业，全球1亿+家庭用户，品牌护城河深
- **ChoreMonster**：家长配置+孩子执行的分工清晰，"家务→积分→礼物"逻辑简单

**做得差的（机会所在）：**
- 没有任何一个产品把**"宠物状态连接现实任务"**做深——宝宝巴士是纯游戏内容，子约/ChoreMonster是积分墙，成长守护是管控工具
- 微信小程序生态里**几乎没有专注4-7岁、宠物+习惯养成**的直接竞品
- 国内市场没有类似ChoreMonster的成熟产品（ChoreMonster面向4-14岁，无宠物）

### PetPal 的差异化空间

```
竞品现状：
  纯游戏内容（宝宝巴士） ← 无现实任务连接
  纯积分墙（子约/ChoreMonster）← 无情感宠物连接
  纯管控工具（成长守护） ← 家长视角，无孩子体验

PetPal 定位：
  宠物 ← 现实任务 → 宠物
  ↑__________________↑
  双向情感连接（竞品空白区）
```

### 关键竞品洞察

**洞察1：宝宝巴士是IP护城河，不是功能护城河**
- 宝宝巴士的优势是200+App矩阵和品牌，不是一个功能做得好
- PetPal聚焦单一场景（宠物+习惯），可以比宝宝巴士做得更深
- 但宝宝巴士是App，需要下载——PetPal小程序即用即走的门槛更低

**洞察2：积分墙模式天然脆弱**
- 子约/ChoreMonster的问题：外部奖励（礼物/玩具）需要家长持续配合
- 一旦家长不配合，奖励链条就断了
- PetPal的解法：奖励是宠物本身的状态，不需要家长介入——孩子自己完成闭环

**洞察3：4-7岁是最空白的目标人群**
- 宝宝巴士偏1-5岁，游戏化太重
- 子约/ChoreMonster偏4岁以上，但无宠物
- PetPal瞄准4-7岁，既有情感认知能力（能理解宠物"饿了"），又不需要复杂任务系统

## 结论

**PetPal V1的核心赌注**：

> 4-7岁孩子的习惯养成，核心不是"告诉孩子应该做什么"，而是让孩子**发自内心想要做**。一只会饿、会摇尾巴、会因为孩子而变得开心的小狗，是这个年龄段最直接的情感连接。

**竞品空白确认**：国内市场没有"宠物+现实任务+习惯养成"的小程序，微信生态里这块几乎是空白。

**成功标准**：上线60天内，D7留存≥40%，且家长每周至少打开1次。

*方案版本：v0.2 | 习惯养成定位 | 目标用户：4-7岁 | 竞品分析已完成*
