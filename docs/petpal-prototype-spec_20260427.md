## 一、原型范围

### 做什么

| 模块 | 优先级 | 说明 |
|-----|--------|------|
| 宠物主界面 | P0 | 状态展示 + 表情动画 |
| 任务打卡 | P0 | 列表 + 打卡 + 审核流程 |
| 积分系统 | P0 | 获取/消耗/余额 |
| 宠物互动 | P0 | 喂食/玩耍/抚摸 |
| 家长模式 | P1 | PIN + 审核 + 周报 |
| 宠物成长 | P2 | 阶段升级（原型先不做） |
| 成就徽章 | P2 | 先不做 |

### 不做什么

- 不做微信登录
- 不做云同步
- 不做多个宠物
- 不做数据导出
- 不做声音/音效
- 不做微信推送通知

## 二、技术选型

### 推荐方案：React 单页应用 + localStorage

**原因**：
- 微信小程序开发需要 AppID，配置复杂
- React 原型可以在浏览器里跑，5分钟出结果
- localStorage 等效模拟小程序本地存储
- 用手机浏览器打开，体验接近小程序

### 技术栈

```
React 18（CDN引入，无需构建）
Tailwind CSS（CDN，手机端样式）
localStorage（数据持久化）
纯 JS（无 TypeScript，原型阶段减少复杂度）
```

### 目录结构

```
/petpal-prototype
  /index.html          # 单文件，包含所有代码
  /SPEC.md             # 本规格书
  /README.md           # 运行说明
```

## 四、宠物状态机

### 衰减规则（核心逻辑）

```js
// 每小时调用一次 decayPet()
// 根据 decaySpeed 设置衰减速度

const DECAY_RULES = {
  relaxed:  { hungerPerHour: 0.5, happinessPerHour: 0.33 },  // 每2小时-1
  standard: { hungerPerHour: 1,    happinessPerHour: 0.5  },  // 每1小时-1
  strict:   { hungerPerHour: 2,    happinessPerHour: 1    }   // 每30分钟-1
}
```

**离线衰减**：用户关闭后再打开，根据 `lastDecayAt` 和当前时间差计算衰减量，确保宠物在离线期间也"饿"。

### 情绪计算

```js
function getMood(hunger, happiness) {
  const avg = (hunger + happiness) / 2;
  if (avg >= 8) return "excited";    // 超级开心 🟢
  if (avg >= 5) return "normal";     // 正常 🟡
  if (avg >= 2) return "sad";        // 难过 🔴
  return "sick";                     // 生病 ⚠️
}
```

### 互动效果

| 动作 | 消耗积分 | 效果 |
|-----|---------|------|
| 喂食 | 1分 | 饱食度 +3（上限10） |
| 玩耍 | 1分 | 快乐值 +2（上限10） |
| 抚摸 | 0分 | 快乐值 +1（上限10） |

**积分不够时**：按钮显示为灰色，点击提示"积分不够了，先去完成任务吧！"

## 六、关键交互细节

### 积分获取

```
任务完成（需家长确认）：
  pending → 家长审核通过 → 积分+1 + 宠物快乐值+1 + 动画奖励

任务完成（无需确认）：
  直接积分+1 + 宠物快乐值+1 + 动画奖励
```

### 动画反馈（CSS 动画，无需 Lottie）

```
宠物开心时：bounce 动画（上下跳动）
积分增加：数字从0跳到目标值（+1）
任务完成：打勾动画 + 星星散落（CSS粒子）
喂食成功：宠物舔嘴 + 饱食度条上涨动画
```

### 状态衰减时机

```
1. 每次打开应用时计算离线时间，执行离线衰减
2. 每60秒检查一次，在线时实时衰减
3. 宠物状态接近0时，界面出现"需要帮助"提示
```

## 七、状态流转图

```
[首次打开] → 设置名字 → [宠物主界面]

[宠物主界面]
  ├→ [今日任务] → 点击打卡 → [待审核] ──→ [家长通过] → +1分 +开心
  │                                      └──→ [家长驳回] → 恢复原状
  ├→ [喂食] → 积分够？ → 是 → 饱食度+3 → 动画反馈
  │              → 否 → 提示"积分不够"
  ├→ [玩耍] → 积分够？ → 是 → 快乐值+2 → 动画反馈
  │              → 否 → 提示"积分不够"
  ├→ [抚摸] → 快乐值+1 → 动画反馈（免费）
  └→ [家长模式] → 输入密码 → 审核/设置

[宠物状态持续衰减]
  ├→ 平均值≥8 → 超级开心表情
  ├→ 平均值5-7 → 正常表情
  ├→ 平均值2-4 → 难过表情
  └→ 平均值<2 → 生病状态（任意互动治愈）
```
